！为什么MySQL要使用B+树
B+树，本质上它是将瘦长的平衡二叉树，改造成一个矮扁的平衡多叉树从而减少IO查询次数，平均查询复杂度仍能保证O(logN),
另外与B树不同的是，除了叶子结点外，其他结点均不存储数据，另外叶子结点之间，用链表方式相连以加速查询。当然视面试官的反应，可以再补充B+树的插入和删除操作。
